Network Diagrams


A Network diagram is a visual representation of a computer or telecommunications network. It shows the components that make up a network and how they interact, including routers, devices, hubs, firewalls, etc. This network diagram shows a local area network (LAN): Depending on its scope and purpose, a network diagram may contain lots of detail or just provide a broad overview. For instance, a diagram of a LAN might could show the IP addresses of individual computers, while the diagram of a MAN (metropolitan area network) could represent buildings or areas with a single node.

A network diagram can be either physical or logical

Logical Network diagrams

A logical network diagram decribes the way information flows through a network. Therefore, logical diagrams typically show subnets (includeing VLAN IDs, Masks, Addresses).Network devices like Routers, Firewalls, and routing protocols.

In the Open Systems Interconnection (OSI) model, logical network diagrams correlate with the information contained in layer 3 (L3). Also known as the “network layer,” L3 is an abstraction layer that deals with packet forwarding through intermediate routers. Level 2 shows the data links between adjacent nodes, while level 1 shows the purely physical layout.


Physical Network diagrams

A physical network disgram shows the actual phuysical arrangement of the components that make up the network, including cables and hardware. Typically, the diagram gives a birds eye view of the network in its physical space like a floor plan.

How are network diagrams used?

With the capacity for showing network components interact, network diagrams can serve a veirity of purposes, including:

  • Planning the structure of a home or professional network
  • Cordinating updates to an existing network
  • Reporting and troubleshooting network problems
  • To Comply to PCI or other requirements
  • As Documentation for external communication, or onboarding etc
  • To keep track of components
  • Sending relivant information to a vendor for a RFP (request for proposal) without disclosing confidential information
  • Selling a relivant proposal to finacial stakeholders
  • Proposing high level, syslog infrastructure changes

What is network topology?

Network topology refers to the arrangment of elements in a network. Like Network diagrams, Network topologies can describe either a physical or logical aspects of a network. Logical topology can also be refered to as signal topology.

Bus Topology

Also known as backbone, linear, or ethernet topology, this type of network is distiguished for having all of the nodes connected by a central medium. The "bus" which has exactly 2 end points.

Bus topologies are easy to configure and require less cable length than some other topologies. However if the central bus breaks down, so does the whole network, and it can be dificult to isolate the problem

Ring Topology

Nodes are connected in a circular configuation, and packets of information are sent through the ring until they reach there destination. Ring topology can outperform those based on the bus topology, and they can easily be reconfiguired to add or remove devices. However, they are still relively vulnerable, since the whole network fails if a single network fails. Also, bandwidth must be shared accross all the devices and connection.

Star Topology

One of the most common topologies, the Star topology consists of a central hub or switch, through which all of the data passes. Along with all the peripheral nodes connected to that central node. Star topology tends to be reliable because individual machines may crash without affecting the rest of the network. But if the central hub or switch fails, none of the connected nodes will be able to access it. Cable cost tend to be higher aswell.

Mesh Topology

There are two types of mesh topologies. In the first, which is called full mesh topologies, each node is directly connected to every other node.

In a partial mesh topology, nodes are only connected to the nodes they connect with the most

Most Networks employ some combination of the topologies to yeild whats called a HYBRID typology For example, the TREE topology is a combination of the bus and mesh topologies.

The logical and physical topology of a particular network may resemble one another, or they may be entirely different. For example a twitsted pair ethernet network exists as a star topology physicaly but follows the bus topology logically.
